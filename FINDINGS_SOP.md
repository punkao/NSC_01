# 📌 Finding: SOP Step-Skip Detection — ทำไมยัง auto ไม่ได้ (สำหรับพูด/สไลด์)

> เก็บไว้เป็น "สิ่งที่เราค้นพบ + งานต่อ" — ไม่ได้ใส่ใน demo เพราะยังไม่น่าเชื่อถือ
> Deliverable ที่ใช้จริง = **F5 (Safety rule + severity)** ซึ่งเชื่อถือได้

---

## เราลองอะไร
เอา narration ที่ VLM (Cosmos) เล่า มาเทียบกับ **7 ขั้นตอน SOP** ว่ามีการ **ข้ามขั้นตอน** ไหม
(เช่น หยิบของเสียจากลัง NG กลับขึ้นสายพาน, เก็บโดยไม่เช็ด)

## เจออะไร — **Data Ceiling: VLM แยกกล่องไม่ออก**
LLM รายงานว่ามีการ "หยิบจากลัง NG กลับขึ้นสายพาน" ที่ 01:53 และ 03:27

**แต่พอเปิดเฟรมจริงดู → worker หยิบจากลัง WAIT (งานปกติ) ไม่ใช่ NG**
→ Cosmos **ระบุกล่องผิด** เพราะ:
- กล่อง NG / WAIT / GOOD เป็นกล่องกระดาษหน้าตาเหมือนกัน
- กระป๋องข้างในเหมือนกัน
- มุมกล้องกดลง (overhead) อ่านป้ายไม่ชัด

→ **ผล = false positive** (แจ้งผิดว่าทำผิดขั้นตอน ทั้งที่ทำปกติ)

## ทำได้ / ทำไม่ได้ (แยกตามขั้นตอน)
| ขั้นตอน SOP | ต้องดู | VLM ทำได้? |
|---|---|---|
| 3 วางลง conveyor · 5 หยิบจาก conveyor · 6 เช็ด | **การกระทำที่เห็นได้** | ✅ ได้ |
| 1,2,7 (WAIT/NG/GOOD) | **แยกกล่อง** | ❌ ไม่ได้ |
| 2 ตรวจสายตา · 4 ตรวจ AI | **การคิด/เครื่อง** | ❌ มองไม่เห็น |

→ ขั้นที่เป็น **การกระทำ** VLM เล่าได้, ขั้นที่พึ่ง **การแยกกล่อง/การคิด** ทำไม่ได้

## ทำไมมันสำคัญ (ไม่ใช่แค่บั๊ก)
SOP-skip ที่มีค่าจริง (R2 หยิบ NG กลับสายพาน, R7 วางโดยไม่ตรวจ, R8 เก็บโดยไม่เช็ด)
**ล้วนพึ่งการแยกกล่อง หรือการตรวจ** → ตรงกับจุดที่ VLM ทำไม่ได้พอดี

## ✅ สิ่งที่เชื่อถือได้แทน (ที่เรา ship)
Safety rules ที่ **ไม่พึ่งกล่อง** VLM ทำได้แม่น → F5:
Near-miss มือใกล้สายพาน · ถอดหมวก/ถุงมือ · ไม่สวม PPE · หน้ากากไม่คลุม · ของแปลกปลอม · คนนอก · ของตก

## 🔮 Future work — ถ้าจะทำ SOP-skip ให้ได้จริง
ต้องแก้ที่ **การแยกกล่องก่อน** (ไม่ใช่แค่ให้ LLM อ่าน text):
1. **ROI คงที่** — กล่องไม่ขยับ → กำหนดพิกัดว่าโซนไหน NG/WAIT/GOOD → เช็คว่ามือเข้าโซนไหน
2. **ติดป้าย/สี/QR** ที่กล่องให้ต่างชัด → VLM/OCR อ่านออก
3. **Object tracking** รายกระป๋อง (track ว่ากระป๋องไปทางไหน)

→ เป็นงานเพิ่ม (vision engineering) ไม่ใช่แค่ prompt LLM
